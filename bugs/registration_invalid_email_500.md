
## Название

Регистрация: при email невалидной длины API возвращает 500 вместо 400

## Шаги воспроизведения

1. Отправить `POST /api/registration/` с телом, где `email` имеет невалидную длину (слишком короткий/слишком длинный).
2. Либо воспроизвести через автотест `test_registration_with_invalid_email_length`:
   - `python -m pytest -vv tests/auth/test_registration.py -k test_registration_with_invalid_email_length`

## Ожидаемый результат

- API возвращает **400 Bad Request**.
- Тело ответа может быть пустым `{}` (если так задумано), но статус должен быть 400.

## Фактический результат

- API возвращает **500 Internal Server Error**.

## Окружение

- Среда: dev
- URL: https://api.dev.ads.ktsf.ru
- Метод: POST /api/registration/

## Приоритет

- **P1**: ожидался **4xx** на невалидный ввод, фактически **500** — это серверная ошибка при клиентской ошибке, влияет на стабильность API и мониторинг.

## Серьёзность

- **High**: ответ **5xx** (внутренняя ошибка сервера).
Примечание:
- Ошибка воспроизводится не на всех значениях: часть невалидных email уже обрабатывается корректно (400), однако для некоторых значений по-прежнему возвращается 500.
